The engineer
behind the terminal.

I take errors as opportunity, not fear. That mindset is what makes me different.

I have been solving infrastructure problems my entire career — maintaining networks in a cybercafe I owned and operated, earning my CCNA and CCNP, repairing systems under pressure. Long before I touched AWS or Terraform, I knew what it cost when systems failed.

Today I build cloud infrastructure that teams can actually own. Terraform. Ansible. Docker. Kubernetes. GitHub Actions. I lead, I finish, and I document everything so nothing is a black box after I leave.

I also build for real clients. The GEORGEL Cancer Foundation — a registered Nigerian NGO partnering with JUTH, the Ministry of Health, and the Nigerian Cancer Society — trusted me to build their public platform. That is the standard I bring to every project.

I co-mentor the DMI DevOps program across Africa because I know what it means to build without a roadmap. I give back because the community gave to me.

If you need an engineer who leads, delivers, and takes full ownership — I am ready.

04 / how I deploy

My deployment
pipeline.

Every project I build follows a real DevOps workflow — from code to cloud.

💻

Local Dev (WSL2)

Write code, Terraform configs, Ansible playbooks on Ubuntu via WSL2

terraform init && terraform plan
→
🐙

GitHub

Push code to GitHub — version control, PR review, CI/CD triggers

git push origin main
→
🏗️

Terraform Provisions

EC2, VPC, security groups, IAM roles spun up automatically

terraform apply -auto-approve
→
⚙️

Ansible Configures

Installs Docker, Nginx, pulls images, sets up services

ansible-playbook site.yml
→
🐳

Docker Runs App

Container spins up with the app, connected to Nginx reverse proxy

docker run -d -p 3000:3000 app
→
☸️

k3s Orchestrates

Kubernetes manages pods, scaling, and service routing

kubectl apply -f deployment.yaml
→
🌍
